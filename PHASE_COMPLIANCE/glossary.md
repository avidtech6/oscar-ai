# Oscar AI Glossary

## Architecture and Development Terms

### FreshVibe 3-Layer Architecture
- **Core Representations**: The fundamental data structures and models that define the application's domain
- **Presentation Modes**: The various ways data can be displayed and interacted with
- **Stable Settings & Preferences**: User preferences and configuration that persist across sessions

### Token-Safe Execution Rules
- **Modular Files**: Code organized into small, focused files for easier maintenance
- **Reversible Changes**: Changes that can be easily undone or rolled back
- **AI Assistance**: AI tools that assist but never rearrange or restructure code
- **Development Constraints**: Rules and limits for safe development practices

### Build Output Size Limits
- **Maximum TypeScript/JavaScript File Size**: 300KB per file
- **Maximum Svelte Component Size**: 200KB per component
- **Maximum JSON File Size**: 500KB per file
- **Maximum GeoJSON File Size**: 1MB per file
- **Maximum SVG Diagram Size**: 400KB per diagram
- **Maximum Worker Script Size**: 400KB per script
- **Maximum WASM Module Size**: 4MB per module
- **Maximum Combined Bundle Size**: 1MB per phase
- **Maximum Lines per Generated File**: 2000 lines

## Report Intelligence System Terms

### Report Types
- **BS5837:2012 Tree Survey**: Arboricultural survey report following British Standard 5837:2012
- **Arboricultural Impact Assessment**: Assessment of tree impact on development projects
- **Arboricultural Method Statement**: Detailed methodology for arboricultural works
- **Tree Inspection Report**: Report on tree health and safety inspections
- **Tree Preservation Order Assessment**: Assessment of trees protected by Tree Preservation Orders
- **Tree Risk Assessment**: Assessment of tree risks and hazards
- **Arboricultural Implication Assessment**: Assessment of arboricultural implications
- **Tree Survey Report**: General tree survey documentation
- **Arboricultural Survey Report**: Comprehensive arboricultural survey documentation

### Report Processing Terms
- **Report Decompiler**: Process of extracting structure and content from documents
- **Structure Map**: Hierarchical representation of document organization
- **Schema Mapping**: Mapping extracted content to report type schemas
- **Schema Updater**: Engine for applying schema updates to existing reports
- **Style Learning**: Analysis of writing style and formatting patterns
- **Report Classification**: Automatic identification of report types
- **Self-Healing**: Automatic detection and fixing of structural inconsistencies
- **Template Generation**: Creating templates from existing reports
- **Compliance Validation**: Validation against regulatory standards
- **Reproduction Testing**: Testing report generation and reproduction consistency

### Report Intelligence Interfaces
- **ReportTypeDefinition**: Interface defining report type specifications
- **DecompiledReport**: Interface for decompiled document structure
- **SchemaMapping**: Interface for content-to-schema mapping
- **StyleProfile**: Interface for learned writing style profiles
- **ClassificationResult**: Interface for report classification results
- **HealingOperation**: Interface for self-healing operations
- **TemplateDefinition**: Interface for report template definitions
- **ComplianceResult**: Interface for compliance validation results
