# **MODULE 9 — CONNECT (CONDENSED VERSION)**

## **9.1 Purpose**

Connect is the **communications + publishing layer** for:

- Conversations
- Campaigns
- Emails
- Social posts
- Blog + SEO
- Landing pages + funnels
- Audience + segments
- Automations
- Analytics

Everything uses the same card + cockpit architecture as Files/Workspace/Projects.

---

## **9.2 Layout**

**Desktop:** Sidebar → Header → Connect Content → Right Panel → Ask Oscar bar

**Mobile:** Header → Content → Bottom bar → Sheets for items

Right panel = back of card.

Left rail = siblings.

Pills = context switch.

## **9.21 ?‑Markers in Editors**

Every block (heading, paragraph, list, table, gallery, code, media) can show a **? icon**.

Clicking it opens a **Help Peek** with:

- Context‑aware questions
- Short explanation
- Link to documentation/templates

Oscar receives exact block context.

---

## **9.22 Settings Peek (Integrations)**

Connect exposes system‑level integrations via a **Settings Peek**:

- Gmail / Google Workspace
- Brevo / Sendinblue
- SMTP
- DKIM / SPF
- Tracking pixels
- Unsubscribe defaults
- Meta / Instagram
- LinkedIn
- X
- TikTok
- YouTube
- WordPress / Ghost / Webflow
- CRM + API keys
- Webhooks

Each setting row has:

- Label
- Input
- **? icon**
- Connection status

---

## **9.23 Inbox Peek + Share‑to‑Oscar**

When configuring providers, Oscar can ask:

> “Do you have a setup/welcome email for this provider?”
> 

If yes:

- Inbox Peek opens
- Emails are pre‑filtered
- Each email has **Share to Oscar**
- Opening an email shows Email View Mode
- Sharing fills settings automatically
- User returns to Settings Peek

---

## **9.24 AI Context Model**

Oscar always knows:

- Domain
- Item
- Module (editor, metadata, settings)
- Active Peek (Settings, Help, Inbox)
- Exact field or block

This enables precise, contextual assistance.
